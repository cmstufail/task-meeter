Question-2: What is the difference between innerHTML, innerText, and textContent ?

১। innerHTML প্রোপার্টি একটি HTML ইলিমেন্টের ভিতরে থাকা টেক্সট, ট্যাগ ও খালি স্থান (স্পেসিং) রিটার্ন করে। ২। innerText প্রোপার্টি শধুমাত্র HTML ইলিমেন্টের ভিতরের টেক্সট রিটার্ন করে। ৩। textContent প্রোপার্টি একটি HTML ইলিমেন্টের ভিতরের খালি জায়গা সহ টেক্সট রিটার্ন করে।

Question-3: What is event delegation in the DOM?

Event deletate হলো একটা টেকনিক যেখানে parent element এর উপর event listener বসিয়ে সেই event কে তার child element গুলোর জন্য handle করা হয়। এতে করে performance ভালো থাকে এবং নতুন element যোগ হলেও আলাদা করে event listener বসাতে হয় না। যদি ডায়নামিক কন্টেন্টের মধ্যে event handler এড করতে চাইলে সেক্ষেত্রে ভালো সিস্টেম হচ্ছে child এর মধ্যে event handler এড না করে parent এর মধ্যে কোনো একটা জায়গায় সেট করে দেওয়া। তাহলে event টা যেখানে ঘটবে সেখানে handle না করে এটাকে delegate করে দেওয়া হয় উপরের কোনো জায়গায়।

Question-4: What is event bubbling in the DOM?

add event listener ৩টা প্যারামিটার নেয়। event type, handler এবং use capture। ৩য় প্যারামিটার অপশনাল এটা ডিক্লেয়ার কর বা না করা, বাই ডিফল্ট false থাকে। প্যারমিটার false মানে bubbling। ধরুন একটা div নিলাম। div এর ভিতরে একটি button নিলাম। দুইটাকে ইভেন্ট লিসেনার দিয়ে সিলেক্ট করে কল করে দিলাম। এখন যদি div এ ক্লিক করি তাহলে div এর ফাংশন কল হবে। অন্যদিকে button এ ক্লিক করলে প্রথমে বাটন এর ফাংশন কল হবে তার পর ডিভ এর ফাংশন হেব। অর্থাৎ নিচ থেকে উপরের দিকে যাওয়া। এই সমস্যা থেকে উত্তরণের জন্য stop.propagation() অথবা stop.immediatePropagation() মেথড ব্যবহার করে বাই ডিফল্ট আচরণ false থেকে true করা হয়। বিষয়টা পিতা-পুত্রের সম্পর্কের মতো। পিতার সম্পত্তিতে পুত্রের অধিকার থাকে, কিন্তু পুত্রের সম্পত্তিতে পিতা অধিকার খাটাতে পারেন ন।
